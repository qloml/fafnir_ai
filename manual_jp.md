# Fafnir AI 開発・強化学習マニュアル v2

このドキュメントは、FafnirのAI（強化学習モデル）の学習、評価、および対戦を管理する全プログラム群の「完全な仕様書」です。
現在実装されているすべての仕様、隠し機能、およびコマンドラインオプションを網羅しています。

---

## 1. ファイル構成とそれぞれの目的

プロジェクト内のAI関連プログラムは、目的ごとに以下の3つのレイヤーに分かれています。

### 1.1. ゲームエンジン・レイヤー（環境）
SB3（機械学習ライブラリ）がゲームをプレイするための「箱」です。
*   **`mppo_ai/rl/fast_engine.py`**
    *   **目的:** 学習を限界まで高速化するための「コア・ゲームロジック」です。
    *   **特徴:** Pythonのクラス等を使わず、Numba (`@njit`) を使用してC言語レベルの速度にコンパイルされています。観測空間の計算、手札の期待値計算、相手の確定手札の追跡など、すべての「計算」はここで行われます。
*   **`mppo_ai/rl/game_env_fast.py`**
    *   **目的:** `fast_engine.py` をSB3が扱える形（Gymnasium環境）に包むラッパーです。
    *   **特徴:** `train.py`（学習）専用です。
*   **`mppo_ai/rl/game_env.py`**
    *   **目的:** Numbaを使わない、純粋なPython実装のGymnasium環境です。
    *   **特徴:** `evaluate.py`（評価）専用です。評価用の各種ボット（GreedyOpponentなど）の実装もここに含まれています。

### 1.2. 学習・評価スクリプト・レイヤー
*   **`mppo_ai/rl/train.py`**
    *   **目的:** モデルを学習させるメインスクリプトです（自己対戦カリキュラム）。
*   **`mppo_ai/rl/evaluate.py`**
    *   **目的:** 学習済みモデルを、強さの異なるテスト用ボットと戦わせて勝率を計測するスクリプトです。

### 1.3. クライアント・レイヤー（オンライン対戦）
*   **`mppo_ai/clients/rl_bot.py`**
    *   **目的:** 実際にWebサーバー (`server.py`) に接続し、他の人間やボットと対戦するプログラムです。
    *   **特徴:** クライアント側でも独自に「相手の確定手札の追跡」と「期待値の計算」を行い、学習時と全く同じ36次元の観測ベクトルを組み立ててAIに渡しています。

## 2. 仕様詳細：AIの認識と報酬

現在のAIが「何を見て」「何を目標にしているか」の完全な仕様です。

### 2.1. 観測空間 (Observation Space - 34次元 / 旧36次元)
AIが毎ターン受け取る情報です（すべて 0.0〜1.0 に正規化）。
※ 現在の最新版(v3)は、ラウンドごとに独立した強化学習を行うため、累計スコアを除外した34次元となっています。旧モデル(v2)は36次元です。

1.  `0-5`: **自分の手札** (6色)
2.  `6-11`: **現在のオファー** (6色)
3.  `12-17`: **ゴミ箱の石** (6色)
4.  `18-23`: **相手の確定手札** (6色)
    *   *仕様:* オークション結果の履歴から、「相手が現在確実に持っている色と数」を推測した値です。
5.  `24`: **相手の不確定な手札の枚数** (1次元)
    *   *仕様:* 相手の合計枚数から確定分を引いた、「中身が分からない伏せ札」の数です。
6.  `25-30`: **自分の確定手札** (6色)
    *   *仕様:* 「相手の視点から見て、自分が持っているとバレている石」の情報です。
7.  `31`: **袋の残り枚数** (旧33)
8.  `32`: **自分が管理人(Caretaker)かどうか** (旧34)
9.  `33`: **自分の手札の期待値スコア** (旧35)
    *   *仕様:* 現時点での確定情報をもとに「もし今ラウンドが終わったら何点か」を計算し、-15〜+60 の範囲を正規化して与えています。

*(※ 旧36次元モデルでは、インデックス31に「自分のスコア」、32に「相手のスコア」が含まれます。)*

### 2.2. 行動空間 (Action Space)
*   **`MultiDiscrete([11] * 6)`**: 6つの色それぞれについて「0〜10個」のどれを出すか同時に決定します。オファーに出ている色や、自分が持っていない色は出せないようマスク（ActionMasker）で強制的に防がれています。

### 2.3. 報酬設計 (Reward Design)
AIが受け取る報酬（プラス/マイナス）は以下の4種類で構成されています。
※ 最新版(v3)では「1ラウンド（1回の得点計算）」を1つのエピソードとして独立して学習します。

1.  **ラウンド終了時の報酬（究極の目標）**
    *   *仕様:* ゴミ箱が上限に達するか、袋が空になってラウンドが終わった際、「(自分がラウンドで得たスコア - 相手が得たスコア) ÷ 30」という差分報酬を与えてエピソードを終了します。これにより、40点の「ゲーム」ではなく、純粋な「ラウンドの得点力」を学習します。
2.  **オークションのスコア差分報酬（ターンの結果）**
    *   入札完了時のチップ増減による「(自分のスコア増加 - 相手のスコア増加) × 0.02」。
3.  **期待値の増減報酬（ポテンシャル報酬）**
    *   *仕様:* 毎ターンの行動前後で「手札の期待値スコア」がどう変化したかを計算し、`(行動後の期待値 - 行動前の期待値) × 0.005` を与えます。
    *   *目的:* 有利な石を拾うことの価値、不要な石を捨てることの価値を即座に教えるためです。
4.  **行動ペナルティ（アクションタックス / Action Tax）**
    *   *仕様:* 入札した石の数の「2乗 × -0.002」の罰則を与えます（例: 1個 = -0.002, 2個 = -0.008, 4個 = -0.032, 5個 = -0.050）。
    *   *目的:* 有利な石を得る見込みがないのに、ただラウンドを早く終わらせるためだけに無駄に石を捨てる（早すぎる手放し）悪癖を防ぎます。特に4個以上の大量投げ捨てを強力に抑制します。

## 3. コマンドライン引数と使用方法

各スクリプトをターミナルで実行する際に使える全オプションです。

### 3.1. 学習 (`mppo_ai/rl/train.py`)
```bash
python mppo_ai/rl/train.py [オプション]
```
| 引数 | デフォルト | 説明 |
| :--- | :--- | :--- |
| `--total-steps` | `500000` | 学習の総ステップ数。 |
| `--score-to-win` | `40` | 勝利に必要なスコア。 |
| `--max-turns` | `500` | 1ゲームの最大ターン数（引き分け打ち切り用）。 |
| `--n-envs` | `8` | 並列実行する環境数。CPUコア数に合わせると高速になります。 |
| `--update-freq` | `20000` | 自己対戦の相手（過去の自分）を更新する頻度。 |
| `--save-freq` | `100000` | `checkpoints/` フォルダに経過モデルを保存する頻度。 |
| `--save-dir` | `mppo_ai/rl/output` | モデルやログを保存するフォルダ。 |
| `--device` | `auto` | `cpu` または `cuda`（GPU使用）。 |
| `--resume` | `None` | 学習を再開したい `.zip` モデルのパス。 |


使用例 : 
```bash
py mppo_ai/rl/train.py --total-steps 1000000 --device cuda --n-envs 16 --resume .\mppo_ai\rl\output\fafnir_final.zip
```

**ブラウザで学習状況を確認する (TensorBoard)**
学習を実行している間（または実行後）に、別のターミナルを開いて以下のコマンドを実行すると、ブラウザ上で学習の進捗（勝率やエピソードの長さなど）をグラフで確認できます。
```bash
# 別のPowerShell（またはコマンドプロンプト）を開き、仮想環境に入ってから実行
venv\Scripts\python.exe -m tensorboard.main --logdir mppo_ai/rl/output/logs
```
実行後、ブラウザで `http://localhost:6006/` にアクセスしてください。

### 3.2. 評価 (`mppo_ai/rl/evaluate.py`)
学習済みモデルの強さを、3種類のテスト用ボット（Random, Greedy, Aggressive）と対戦させて計測します。
```bash
python mppo_ai/rl/evaluate.py --model mppo_ai/rl/output/fafnir_final.zip [オプション]
```
| 引数 | デフォルト | 説明 |
| :--- | :--- | :--- |
| `--model` | (必須) | 評価したいモデルのパス。 |
| `--games` | `200` | 各ボットと何回対戦させるか。 |
| `--score-to-win` | `40` | 勝利に必要なスコア。 |
| `--max-turns` | `500` | 1ゲームの最大ターン数。 |
| `--deterministic` | 指定なし | このフラグをつける（Trueにする）と、AIが「最も確率の高い手」を常に選ぶようになります（ブレがなくなる）。 |

### 3.3. サーバー起動 (`server_0424.py`)
オンライン対戦やGUIツールを使用する際は、まずサーバーを立ち上げる必要があります。
```bash
uvicorn server_0424:socket_app --host 0.0.0.0 --port 8765
```
※ バグ修正版の `server_0424.py` を使用します。

### 3.4. クライアント関連 (`mppo_ai/clients/` フォルダ)
サーバー (`server_0424.py`) を立ち上げた状態で、プレイヤーや観戦者として接続するためのプログラム群です。

#### ① `mppo_ai/clients/rl_bot_v2.py` / `rl_bot_v3.py` (強化学習AIクライアント)
サーバーに学習済みAIを接続させます。
※ `v2`は旧モデル(36次元)用、`v3`は新モデル(34次元・ラウンド学習版)用です。
```bash
python mppo_ai/clients/rl_bot_v3.py --model mppo_ai/rl/output/fafnir_final.zip [オプション]
```
| 引数 | デフォルト | 説明 |
| :--- | :--- | :--- |
| `--model` | (必須) | 使用するモデルのパス。 |
| `--url` | `http://127.0.0.1:8765` | 接続先のサーバーURL。 |
| `--room` | `room1` | 入室するルーム名。 |
| `--name` | `RLBot` | 画面に表示されるAIの名前。 |
| `--deterministic` | `1` | `1`なら最強の手を打ち、`0`なら少し確率的に揺らぎのある手を打ちます。 |
| `--think-delay` | `0.05` | AIが手を打つまでの待機時間（秒）。 |

#### ② `mppo_ai/clients/pimc_bot_v2.py` / `pimc_bot_v3.py` (探索付き強化学習AIクライアント - PIMC)
サーバーに学習済みAIを接続させるプログラムですが、単純なニューラルネットワークの出力だけでなく、実行時に**PIMC (Perfect Information Monte Carlo) 探索**を行ってより強力な手を打ちます。
※ `v2`は旧モデル(36次元)用、`v3`は新モデル(34次元・ラウンド学習版)用です。
※ **注意:** このスクリプトはオンライン対戦クライアントとしてのみ機能し、モデル自体の「学習（重みの更新）」は一切行いません。学習は `train.py` の役割です。

**【PIMC探索アルゴリズムの解説】**
Fafnirは「相手の伏せ札」という不完全情報を持つゲームです。PIMCボットでは以下のように探索を行います。
1. **決定化 (Determinization):** 「相手の確定手札」と「袋の残り石」の情報をもとに、相手の伏せ札のランダムな組み合わせパターンを複数生成し、一時的に「完全情報ゲーム」と仮定します。
2. **候補手の抽出:** 学習済みPPOモデルを使って、現在の盤面から複数の「候補となる手（最も確率の高い手＋確率的な手）」をサンプリングします。
3. **1手先読みと評価:** 高速シミュレータ (`fast_engine.py`) を使い、各候補手を打った1手先の盤面を作ります。その後、PPOの「価値ネットワーク (Value Network)」を使用してその盤面の有利さを数値化します。
4. **最善手の選択:** 複数の決定化パターンすべてをシミュレーションし、平均して最も評価が高かった候補手を最終的な行動として送信します。

```bash
python mppo_ai/clients/pimc_bot_v3.py --model mppo_ai/rl/output/fafnir_final.zip [オプション]
```
| 引数 | デフォルト | 説明 |
| :--- | :--- | :--- |
| `--model` | (必須) | 使用するモデルのパス。 |
| `--url` | `http://127.0.0.1:8765` | 接続先のサーバーURL。 |
| `--room` | `room1` | 入室するルーム名。 |
| `--name` | `PIMC_Bot` | 画面に表示されるAIの名前。 |
| `--search-time` | `0.2` | 1手あたりの探索に使う時間（秒）。 |
| `--candidates` | `4` | 1つの決定化パターンにつきサンプリングする候補手の数。 |
| `--det-batch` | `4` | 1回のGPU評価にまとめる決定化パターン数。大きいほどGPU効率が上がりますが、時間制限の粒度が粗くなります。 |

#### ③ `mppo_ai/clients/human_cli.py` (CUI人間用クライアント)
ターミナル上で人間がプレイするためのクライアントです。
```bash
python mppo_ai/clients/human_cli.py --name Player1 --room room1
```

#### ④ `mppo_ai/clients/ai_bot_sample.py` (ルールベースAIクライアント)
ランダムや簡単なルールで動くサンプルボットです。RLボットの対戦相手としてテスト起動するのに便利です。
```bash
python mppo_ai/clients/ai_bot_sample.py --name DummyBot --room room1
```

#### ⑤ `mppo_ai/clients/spectator_gui.py` (観戦用GUI)
Pygameを使用したデスクトップ用の観戦画面です。対戦には参加せず、現在の盤面や両者の手札（公開情報のみ）をグラフィカルに表示します。
```bash
python mppo_ai/clients/spectator_gui.py --room room1
```

#### ⑥ `mppo_ai/clients/web_gui/` (ブラウザ用UI)
ブラウザ上で人間がプレイするためのWebフロントエンド一式（HTML/CSS/JS）です。`index.html` をブラウザで開くことで、サーバーに接続してGUIでプレイできます。
